## ٢. آلية الربط بين المقالات ثنائية اللغة

كل مقالة تملك في مقدمتها (`front matter`) مفتاحين أساسيين:
- `slug`: اسم فريد يربط النسختين العربية والإنجليزية **متطابق تماماً** في المقالتين.
- `image`: اسم ملف الصورة داخل مجلد `assets/images/ar/` أو `en/` حسب اللغة.

**مثال مقال عربي** `_ar/ai/2025-06-01-ثورة-الذكاء.md`:
```yaml
---
layout: post
title: "ثورة الذكاء الاصطناعي في 2025"
date: 2025-06-01
category: ai
lang: ar
slug: ai-revolution-2025
image: ai-revolution.jpg
---
محتوى المقال...
```

**نظيره الإنجليزي** `_en/ai/2025-06-01-ai-revolution.md`:
```yaml
---
layout: post
title: "The AI Revolution of 2025"
date: 2025-06-01
category: ai
lang: en
slug: ai-revolution-2025
image: ai-revolution.jpg
---
Article content...
```

بهذا الشكل، في صفحة المقال نستطيع الوصول إلى الترجمة عن طريق البحث عن مقالة من المجموعة الأخرى بنفس `slug`، وعرض رابط "English / العربية".
كما أن الصورة تُجلَب تلقائياً من المجلد المناسب:
`/assets/images/{{ page.lang }}/{{ page.image }}`.

### `_layouts/post.html`

```html

    {% endif %}
    <h1>{{ page.title }}</h1>
    <div class="post-meta">
      <time datetime="{{ page.date | date: '%Y-%m-%d' }}">{{ page.date | date: "%B %d, %Y" }}</time>
      <span class="post-category">{{ page.category | replace: '-', ' ' | capitalize }}</span>

    {% else %}

        {% if page.lang == 'ar' %}Read in English{% else %}اقرأ بالعربية{% endif %}

    {% endif %}
  </header>

    {{ content }}
  </div>
</article>
```

## ٦. سكريبت تبديل الوضع الداكن (تلقائي + يدوي)

`assets/js/theme.js`:
```javascript
(function () {
  const html = document.documentElement;
  const toggleBtn = document.getElementById('theme-toggle');

  const getPreferredTheme = () => {
    const stored = localStorage.getItem('theme');
    if (stored) return stored;
    return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
  };

  const applyTheme = (theme) => {
    html.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
  };

  // تطبيق أولي
  applyTheme(getPreferredTheme());

  // استماع لتغيير إعدادات النظام
  window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', (e) => {
    if (!localStorage.getItem('theme')) {
      applyTheme(e.matches ? 'dark' : 'light');
    }
  });

  // التبديل اليدوي
  if (toggleBtn) {
    toggleBtn.addEventListener('click', () => {
      const current = html.getAttribute('data-theme');
      const next = current === 'dark' ? 'light' : 'dark';
      applyTheme(next);
    });
  }
})();
```

## ٨. إضافات مفيدة اقترحتها

- **RSS تلقائي** للمقالات من آخر 10 منشورات (بفضل `jekyll-feed`).
- **خريطة موقع XML** لتحسين محركات البحث (`jekyll-sitemap`).
- **مُحسّنات SEO** عبر `jekyll-seo-tag`.
- **نظام تعليقات** يمكن إضافته بسهولة باستخدام Disqus (سأدمج مقتطفاً بسيطاً في `post.html`).
- **زر "شارك"** لشبكات التواصل (تويتر، فيسبوك) – سأضيف كوداً اختيارياً.
- **تحميل كسول** للصور (`loading="lazy"`) لتحسين الأداء.
- **تصميم متجاوب** بالكامل (grid + flex).
- **دعم اتجاهي RTL/LTR** تلقائي حسب اللغة.

(سأضمّن تعليقات في القالب لتسهيل إضافة Disqus إذا أردت).

## ٩. خطوات النشر

1. أنشئ مستودعاً باسم `GT-NEWSTECH` في حسابك على GitHub.
2. انسخ جميع الملفات أعلاه مع الحفاظ على الهيكل.
3. تأكد من وضع بعض المقالات العربية والإنجليزية في `_ar` و `_en` مع رابط `slug` متطابق للترجمة.
4. ارفع التغييرات (push) إلى الفرع `main`.
5. انتقل إلى إعدادات المستودع > Pages، وتأكد من أن المصدر هو `Deploy from a branch` واختر `main` والمجلد `/ (root)`. ستجد موقعك على `https://username.github.io/GT-NEWSTECH/`.

لإضافة مقال جديد:
- ضع ملف `.md` في المجلد الصحيح (`_ar` أو `_en`).
- تأكد من الحقول: `slug`, `category`, `image`, `lang`.
- ارفع الصورة المناسبة في `assets/images/ar/` أو `en/`.
- سيظهر المقال تلقائياً في الصفحة الرئيسية وفي صفحة التصنيف المناسبة.

# دليل تغيير النطاق — GT-NEWSTECH

> هذا الدليل يشرح الخطوات التقنية الكاملة للانتقال من نطاق GitHub Pages
> إلى نطاق مخصص مثل `gt-newstech.com`.

---

## قبل البدء — ما تحتاجه

- [ ] نطاق مشترى وجاهز (من Namecheap أو GoDaddy أو Cloudflare أو غيرها)
- [ ] صلاحية الوصول إلى إعدادات DNS للنطاق
- [ ] صلاحية الوصول إلى إعدادات مستودع GitHub

---

## الخطوة 1 — تعديل `_config.yml`

افتح ملف `_config.yml` في جذر المشروع وعدّل السطرين التاليين:

**قبل التعديل:**
```yaml
url:     "https://SalehGNUTUX.github.io"
baseurl: "/GT-NEWSTECH"
```

**بعد التعديل:**
```yaml
url:     "https://gt-newstech.com"
baseurl: ""
```

> **ملاحظة:** `baseurl` يصبح فارغاً لأن النطاق المخصص يُشير مباشرة لجذر الموقع،
> خلافاً لـ GitHub Pages التي تضع الموقع داخل مسار `/GT-NEWSTECH/`.

---

## الخطوة 2 — إنشاء ملف `CNAME`

أنشئ ملفاً باسم `CNAME` (بدون امتداد) في جذر المشروع، ويحتوي على سطر واحد فقط:

```
gt-newstech.com
```

> **مهم:** لا تضع `https://` ولا `www.` — اسم النطاق فقط.
> GitHub Pages يقرأ هذا الملف ليعرف النطاق المرتبط بالمستودع.

---

## الخطوة 3 — إعداد DNS عند مزود النطاق

سجّل الدخول إلى لوحة تحكم مزود النطاق وأضف السجلات التالية:

### إذا أردت `gt-newstech.com` (بدون www) — سجلات A

أضف **4 سجلات A** تشير إلى خوادم GitHub Pages:

| النوع | الاسم | القيمة |
|---|---|---|
| A | `@` | `185.199.108.153` |
| A | `@` | `185.199.109.153` |
| A | `@` | `185.199.110.153` |
| A | `@` | `185.199.111.153` |

### إضافة www (اختياري لكن موصى به) — سجل CNAME

| النوع | الاسم | القيمة |
|---|---|---|
| CNAME | `www` | `salehgnutux.github.io` |

> **تنبيه:** تغييرات DNS قد تأخذ من **30 دقيقة حتى 48 ساعة** لتنتشر عالمياً.
> يمكنك التحقق من الانتشار عبر: `dig gt-newstech.com` في الطرفية.

---

## الخطوة 4 — إعداد GitHub Pages

1. اذهب إلى مستودع `GT-NEWSTECH` على GitHub
2. انقر **Settings** (الإعدادات)
3. من القائمة الجانبية اختر **Pages**
4. في حقل **Custom domain** أدخل: `gt-newstech.com`
5. انقر **Save**
6. انتظر دقيقتين ثم فعّل **Enforce HTTPS** ✓

> GitHub سيتحقق تلقائياً من سجلات DNS قبل تفعيل HTTPS.
> إن لم يتحقق فوراً، انتظر انتشار DNS ثم أعد المحاولة.

---

## الخطوة 5 — رفع التغييرات

```bash
cd /مسار/GT-NEWSTECH

git add _config.yml CNAME
git commit -m "config: الانتقال إلى النطاق المخصص gt-newstech.com"
git push origin main
```

بعد اكتمال GitHub Actions (دقيقة إلى دقيقتين) يصبح الموقع متاحاً على النطاق الجديد.

## الخطوة 6 — التحقق من عمل كل شيء

بعد انتشار DNS واكتمال البناء، تحقق من النقاط التالية:

### روابط الموقع
```
https://gt-newstech.com/              ← الصفحة الرئيسية
https://gt-newstech.com/ar/           ← الصفحة العربية
https://gt-newstech.com/en/           ← الصفحة الإنجليزية
https://gt-newstech.com/sitemap.xml   ← خريطة الموقع
https://gt-newstech.com/feed.xml      ← RSS Feed
```

### التحقق من HTTPS
- يجب أن يظهر قفل 🔒 في المتصفح
- أي زيارة لـ `http://` تُحوَّل تلقائياً لـ `https://`

### التحقق من Redirect القديم
افتح في المتصفح:
```
https://SalehGNUTUX.github.io/GT-NEWSTECH/
```
يجب أن يُحوَّل تلقائياً (301) إلى:
```
https://gt-newstech.com/
```

## الخطوة 7 — تحديث Google Search Console

إذا كنت قد أضفت الموقع مسبقاً إلى Google Search Console:

1. اذهب إلى [search.google.com/search-console](https://search.google.com/search-console)
2. أضف **Property جديد** بالنطاق الجديد `gt-newstech.com`
3. تحقق من الملكية (عبر DNS أو ملف HTML)
4. أرسل `sitemap.xml` الجديد:
   ```
   https://gt-newstech.com/sitemap.xml
   ```
5. استخدم أداة **Change of Address** في الـ Property القديم لإخبار Google بالانتقال

## مشكلات شائعة وحلولها

### الموقع يعمل لكن الأصول (CSS/JS/صور) مكسورة
**السبب:** نسيت تفريغ `baseurl` في `_config.yml`
**الحل:** تأكد أن `baseurl: ""` وليس `baseurl: "/GT-NEWSTECH"`

---

### GitHub لا يقبل تفعيل HTTPS
**السبب:** سجلات DNS لم تنتشر بعد
**الحل:** انتظر 24-48 ساعة ثم أعد المحاولة من Settings → Pages

---

### `www.gt-newstech.com` لا يعمل
**السبب:** لم تضف سجل CNAME لـ www
**الحل:** أضف سجل CNAME كما في الخطوة 3، أو أضف سجلات A لـ `www` أيضاً

---

### الموقع القديم لا يُحوَّل
**السبب:** GitHub يفعّل الـ redirect فقط بعد إضافة ملف CNAME ورفعه
**الحل:** تأكد أن ملف CNAME موجود في المستودع وتم رفعه

---

## إذا أردت Cloudflare (موصى به)

Cloudflare يوفر طبقة إضافية من الحماية والأداء:

1. أضف نطاقك في [cloudflare.com](https://cloudflare.com)
2. غيّر Nameservers عند مزود النطاق لـ Cloudflare
3. أضف سجلات A لـ GitHub Pages في Cloudflare DNS
4. **مهم:** اضبط Proxy Status على **DNS only** (السحابة الرمادية) وليس Proxied
   > GitHub Pages لا يدعم الـ Proxy المباشر من Cloudflare مع HTTPS

## ملخص التغييرات المطلوبة

```
_config.yml        ← عدّل url و baseurl
CNAME              ← أنشئ الملف بالنطاق الجديد
GitHub Settings    ← أضف النطاق وفعّل HTTPS
DNS                ← أضف سجلات A و CNAME
git push           ← ارفع التغييرات
```

**المجموع: تعديل ملفين + إعدادات DNS + 5 دقائق عمل فعلي.**

## ٢. إنشاء مقال جديد — خطوة بخطوة

### الخطوة 1 — تسمية الملف

اسم الملف يتبع هذا النمط الثابت:

```
YYYY-MM-DD-وصف-مختصر.md
```

| الجزء | الشرح |
|---|---|
| `YYYY-MM-DD` | تاريخ المقال بالأرقام (مثل `2026-05-20`) |
| `وصف-مختصر` | كلمات قليلة تصف الموضوع، مفصولة بـ `-` |
| `.md` | امتداد Markdown |

**أمثلة صحيحة:**
```
_ar/ai/2026-05-20-مقارنة-نماذج-الذكاء.md
_en/ai/2026-05-20-ai-models-comparison.md
```

---

### الخطوة 2 — كتابة رأس المقال (Front Matter)

في بداية كل ملف `.md` يوجد قسم محاط بـ `---` يُسمى **Front Matter**، وهو إعدادات المقال.

**المقال العربي** `_ar/ai/2026-05-20-مقارنة-نماذج-الذكاء.md`:

```markdown
---
layout: post
title: "مقارنة أبرز نماذج الذكاء الاصطناعي في 2026"
date: 2026-05-20
category: ai
lang: ar
slug: ai-models-comparison-2026
image: ai-models-ar.jpg
author: GNUTUX
tags: [ذكاء اصطناعي, نماذج, مقارنة]
excerpt: "نستعرض في هذا المقال أبرز نماذج اللغة الكبيرة المتاحة في 2026 ونقارن بينها من حيث الأداء والتكلفة وحالات الاستخدام."
---

محتوى المقال هنا...
```

**المقال الإنجليزي النظير** `_en/ai/2026-05-20-ai-models-comparison.md`:

```markdown
---
layout: post
title: "Comparing the Top AI Models of 2026"
date: 2026-05-20
category: ai
lang: en
slug: ai-models-comparison-2026
image: ai-models-en.jpg
author: GNUTUX
tags: [AI, models, comparison]
excerpt: "We review the most prominent large language models available in 2026, comparing performance, cost, and use cases."
---

Article content here...
```

> **⚠️ القاعدة الأهم:** قيمة `slug` يجب أن تكون **متطابقة حرفياً** في المقالين. هذا هو الرابط الوحيد الذي يُعرِّف الموقعَ أن المقالين نسخة من بعضهما، ويُظهر زر "اقرأ بالعربية / Read in English".

## ٤. الصور

### المواصفات المطلوبة

| المعيار | القيمة |
|---|---|
| **المقاس** | 1200 × 630 بكسل (نسبة 16:9) |
| **الصيغة** | JPG للصور، PNG للشعارات |

### مكان وضع الصور

```
assets/images/ar/اسم-الصورة.jpg   ← صورة المقال العربي
assets/images/en/اسم-الصورة.jpg   ← صورة المقال الإنجليزي
```

- يمكن أن تكون **نفس الصورة** بنفس الاسم في المجلدين.
- يمكن أن تكون **صورتان مختلفتان** (مثلاً نسخة بنص عربي وأخرى بنص إنجليزي).
- في Front Matter، اكتب **اسم الصورة فقط** بدون المسار:

```yaml
image: اسم-الصورة.jpg
```

### مقال بدون صورة

احذف سطر `image:` كلياً. سيظهر تلقائياً خلفية بلون القسم مع أيقونة.

## ٥. روابط الأفيلييت

### أولاً — تفعيل الإفصاح التلقائي

أضف هذا السطر في الـ Front Matter لأي مقال يحتوي روابط أفيلييت:

```yaml
affiliate: true
```

سيظهر تلقائياً شريط إفصاح ذهبي اللون أعلى المقال يُعلم القارئ بوجود روابط أفيلييت — وهو مطلوب قانونياً ومن سياسة Google.

---

### ثانياً — كتابة رابط الأفيلييت داخل المقال

```markdown
[اسم المنتج أو الخدمة](https://رابط-الأفيلييت.com){: .aff-link rel="nofollow sponsored" target="_blank"}
```

| العنصر | الغرض |
|---|---|
| `.aff-link` | تنسيق بصري مميز للرابط (ذهبي مع سهم ↗) |
| `rel="nofollow sponsored"` | يُخبر محركات البحث أن الرابط تجاري |
| `target="_blank"` | يفتح في نافذة جديدة |

## ٦. نشر مقال في أكثر من قسم

بعض المقالات تصلح لأكثر من قسم — مثل مقال عن دعم ARM في لينكس يصلح لـ `tech-news` و`gnulinux` معاً.

### الطريقة: حقل `also_in`

- `category` ← القسم **الرئيسي** الذي يُحفظ فيه الملف ويحدد لون البطاقة
- `also_in` ← قائمة الأقسام **الإضافية** التي يظهر فيها المقال أيضاً

```yaml
---
layout: post
title: "رقائق ARM وتأثيرها على لينكس"
date: 2026-05-20
category: tech-news        # القسم الرئيسي
lang: ar
slug: arm-linux-impact
also_in: [gnulinux, ai]   # يظهر أيضاً في هذين القسمين
---
```

> - الملف يُوضع دائماً في مجلد القسم **الرئيسي**: `_ar/tech-news/`
> - لا حاجة لنسخ الملف — يظهر تلقائياً في جميع الأقسام المدرجة في `also_in`
> - شارة القسم الإضافي تظهر على بطاقة المقال

## ٧. جميع خيارات Front Matter

```yaml
---
# ─── إلزامية ───────────────────────────────
layout: post                        # لا تغيّر هذا
title: "عنوان المقال"               # العنوان الكامل
date: 2026-05-20                    # تاريخ النشر
category: ai                        # القسم (من الجدول أعلاه)
lang: ar                            # ar أو en
slug: my-unique-slug                # معرّف فريد، نفسه في النسختين

# ─── موصى بها ──────────────────────────────
image: my-image.jpg                 # صورة الترويسة (في assets/images/ar/ أو en/)
author: GNUTUX                      # اسم الكاتب
excerpt: "ملخص قصير للمقال..."     # يظهر في القوائم وفي SEO

# ─── اختيارية ──────────────────────────────
tags: [وسم1, وسم2, وسم3]           # وسوم للمقال
also_in: [foss, gnulinux]          # أقسام إضافية يظهر فيها المقال (اختياري)
affiliate: true                     # أضفها فقط إذا كان المقال يحتوي روابط أفيلييت
description: "نص لـ SEO مخصص"     # إن تركته يستخدم excerpt تلقائياً
---
```

## ٧. قواعد الـ slug

| القاعدة | مثال صحيح | مثال خاطئ |
|---|---|---|
| أحرف لاتينية فقط | `ubuntu-review-2026` | `مراجعة-أوبونتو` |
| أحرف صغيرة فقط | `arch-linux-guide` | `Arch-Linux-Guide` |
| شرطة `-` بدل المسافات | `best-keyboards-2026` | `best keyboards 2026` |
| فريد لكل مقال | ✅ | لا تكرر slug موجود |
| **متطابق في AR و EN** | كلاهما `ubuntu-review-2026` | ❌ مختلفان |

## ٨. نشر المقال على GitHub

```bash
# 1. انتقل لمجلد المشروع
cd /مسار/GT-NEWSTECH

# 2. أضف الملفات الجديدة
git add _ar/ _en/ assets/images/

# 3. سجّل التغييرات
git commit -m "add: مقال — عنوان المقال هنا"

# 4. ارفع للموقع
git push origin main
```

الموقع يتحدث خلال **دقيقة إلى دقيقتين** تلقائياً عبر GitHub Actions.

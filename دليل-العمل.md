## ٢. إضافة مقال جديد

### الخطوة 1 — إنشاء ملف المقال العربي

أنشئ ملفاً في المجلد المناسب داخل `_ar/` باسم بهذا الشكل:

```
YYYY-MM-DD-عنوان-مختصر.md
```

**مثال:** `_ar/ai/2026-05-15-مقارنة-نماذج-الذكاء.md`

**محتوى الملف:**

```markdown
---
layout: post
title: "عنوان المقال بالكامل"
date: 2026-05-15
category: ai
lang: ar
slug: ai-models-comparison-2026
image: ai-comparison.jpg
author: GNUTUX
tags: [ذكاء اصطناعي, مقارنة, نماذج]
excerpt: "ملخص قصير للمقال يظهر في القوائم (جملة أو جملتان)."
---

## العنوان الأول

محتوى المقال هنا بصيغة Markdown...
```

---

### الخطوة 2 — إنشاء ملف المقال الإنجليزي (النظير)

أنشئ ملفاً في المجلد المقابل داخل `_en/`:

**مثال:** `_en/ai/2026-05-15-ai-models-comparison.md`

```markdown
---
layout: post
title: "AI Models Comparison 2026"
date: 2026-05-15
category: ai
lang: en
slug: ai-models-comparison-2026
image: ai-comparison.jpg
author: GNUTUX
tags: [AI, comparison, models]
excerpt: "A short summary of the article for listing pages."
---

## First Heading

Article content in Markdown...
```

> **⚠️ القاعدة الذهبية:** قيمة `slug` يجب أن تكون **متطابقة تماماً** في المقالين العربي والإنجليزي. هذا ما يربطهما ويظهر زر تبديل اللغة.

## ٤. إضافة الصور

### المقاسات الموصى بها
- **صورة الترويسة:** `1200 × 630` بكسل (نسبة 16:9)
- **الصيغة:** JPG (للصور التقنية) أو PNG (للشعارات)
- **الحجم:** أقل من 300 كيلوبايت للأداء الجيد

### مكان وضع الصور
```
assets/images/ar/اسم-الصورة.jpg   ← للمقال العربي
assets/images/en/اسم-الصورة.jpg   ← للمقال الإنجليزي
```

يمكن أن تكون الصورتان مختلفتين (مثلاً نسخة عربية ونسخة إنجليزية) أو نفس الصورة بنفس الاسم في كلا المجلدين.

في ملف المقال، أشر إلى اسم الصورة فقط (بدون المسار):
```yaml
image: اسم-الصورة.jpg
```

---

## ٥. المقال بدون صورة

إذا لم يكن للمقال صورة، احذف سطر `image:` كلياً. سيظهر مكان الصورة خلفية ملونة بلون القسم مع أيقونة.

---

## ٦. روابط الأفيلييت في المقال

إذا كان المقال يحتوي على روابط أفيلييت، أضف هذا السطر في الـ front matter:

```yaml
affiliate: true
```

سيظهر تلقائياً شريط إفصاح قانوني أعلى المقال.

لكتابة رابط أفيلييت داخل المقال:

```markdown
[اسم المنتج](https://رابط-الأفيلييت.com){: .aff-link rel="nofollow sponsored" target="_blank"}
```

---

## ٧. خيارات إضافية في front matter

```yaml
# مقال مميز يظهر أولاً
featured: true

# كاتب مختلف عن الافتراضي
author: "اسم الكاتب"

# وسوم للمقال
tags: [وسم1, وسم2, وسم3]

# تعطيل جدول المحتويات
toc: false

# وصف بديل لـ SEO (اختياري، يستخدم excerpt افتراضياً)
description: "وصف مخصص لـ SEO بين 120 و160 حرف"
```

## ٨. نشر المقال على GitHub

بعد إنشاء الملفات وإضافة الصور:

```bash
cd /مسار/GT-NEWSTECH

git add .
git commit -m "add: مقال — عنوان المقال"
git push origin main
```

سيُنشر المقال خلال دقيقتين تلقائياً عبر GitHub Actions.

## ٩. قواعد تسمية الـ slug

- **باللاتينية فقط** (لا عربية)
- **أحرف صغيرة** فقط
- **شرطة `-` بدل المسافات**
- فريد لكل مقال

**صحيح:** `ubuntu-2504-review`
**خاطئ:** `Ubuntu 25.04 Review` أو `مراجعة-أوبونتو`

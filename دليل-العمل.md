## ٢. إنشاء مقال جديد — خطوة بخطوة

### الخطوة 1 — تسمية الملف

اسم الملف يتبع هذا النمط الثابت:

```
YYYY-MM-DD-وصف-مختصر.md
```

| الجزء | الشرح |
|---|---|
| `YYYY-MM-DD` | تاريخ المقال بالأرقام (مثل `2026-05-20`) |
| `وصف-مختصر` | كلمات قليلة تصف الموضوع، مفصولة بـ `-` |
| `.md` | امتداد Markdown |

**أمثلة صحيحة:**
```
_ar/ai/2026-05-20-مقارنة-نماذج-الذكاء.md
_en/ai/2026-05-20-ai-models-comparison.md
```

في بداية كل ملف `.md` يوجد قسم محاط بـ `---` يُسمى **Front Matter**، وهو إعدادات المقال.

## ٤. الصور

### المواصفات المطلوبة

| المعيار | القيمة |
|---|---|
| **المقاس** | 1200 × 630 بكسل (نسبة 16:9) |
| **الصيغ المدعومة** | `jpg, jpeg, png, webp, avif, gif, svg` |
| **الصيغ غير المدعومة** | `heic, heif, tiff, bmp` → تُحوَّل تلقائياً إلى JPEG عبر لوحة التحكم |
| **الحجم الأقصى** | 20 MB عبر لوحة التحكم |
| **الاسم** | أحرف لاتينية وأرقام وشرطات فقط |

### رفع الصور — من لوحة التحكم (الأسهل)

1. افتح **مدير الصور** في لوحة التحكم
2. اختر اللغة: AR أو EN
3. اسحب الصورة وأفلتها **أو** انقر لتصفح ملفات الجهاز
4. **أو:** اضغط **استيراد من الجهاز** لاستيراد صورة من أي مكان مع اختيار الوجهة (AR / EN / كليهما دفعة واحدة)
5. بعد الرفع: انسخ اسم الصورة وضعه في حقل `image:` في المحرر

### مكان الصور يدوياً

```
assets/images/ar/اسم-الصورة.jpg   ← صورة المقال العربي
assets/images/en/اسم-الصورة.jpg   ← صورة المقال الإنجليزي
```

في Front Matter، اكتب **اسم الصورة فقط** بدون المسار:

```yaml
image: my-photo.jpg
```

### مقال بدون صورة

احذف سطر `image:` كلياً. يظهر شعار الموقع تلقائياً مكانها.

## ٨. نشر مقال في أكثر من قسم

بعض المقالات تصلح لأكثر من قسم — مثل مقال عن دعم ARM في لينكس يصلح لـ `tech-news` و`gnulinux` معاً.

### الطريقة: حقل `also_in`

- `category` ← القسم **الرئيسي** الذي يُحفظ فيه الملف ويحدد لون البطاقة
- `also_in` ← قائمة الأقسام **الإضافية** التي يظهر فيها المقال أيضاً

```yaml
---
layout: post
title: "رقائق ARM وتأثيرها على لينكس"
date: 2026-05-20
category: tech-news        # القسم الرئيسي
lang: ar
slug: arm-linux-impact
also_in: [gnulinux, ai]   # يظهر أيضاً في هذين القسمين
---
```

> - الملف يُوضع دائماً في مجلد القسم **الرئيسي**: `_ar/tech-news/`
> - لا حاجة لنسخ الملف — يظهر تلقائياً في جميع الأقسام المدرجة في `also_in`
> - شارة القسم الإضافي تظهر على بطاقة المقال

## ٩. جميع خيارات Front Matter

```yaml
---
# ─── إلزامية ───────────────────────────────
layout: post                        # لا تغيّر هذا
title: "عنوان المقال"               # العنوان الكامل
date: 2026-05-20 14:30:00          # تاريخ + وقت النشر (يحدد الترتيب)
category: ai                        # القسم (من الجدول أعلاه)
lang: ar                            # ar أو en
slug: my-unique-slug                # معرّف فريد، نفسه في النسختين

# ─── موصى بها ──────────────────────────────
image: my-image.jpg                 # صورة الترويسة (في assets/images/ar/ أو en/)
author: GNUTUX                      # اسم الكاتب
excerpt: "ملخص قصير للمقال..."     # يظهر في القوائم وفي SEO

# ─── اختيارية ──────────────────────────────
tags: [وسم1, وسم2, وسم3]           # وسوم للمقال
also_in: [foss, gnulinux]          # أقسام إضافية يظهر فيها المقال (اختياري)
affiliate: true                     # أضفها فقط إذا كان المقال يحتوي روابط أفيلييت
description: "نص لـ SEO مخصص"     # إن تركته يستخدم excerpt تلقائياً
---
```

> **ملاحظة الوقت:** إذا نشرت أكثر من مقال في نفس اليوم، استخدم وقتاً مختلفاً (`14:30:00`) لكل مقال لضمان ظهوره في المكان الصحيح في "أحدث المقالات". لوحة التحكم تُضيف الوقت تلقائياً.

> **ملاحظة المنطقة الزمنية:** لوحة التحكم تحفظ التاريخ بـ UTC الفعلي بصرف النظر عن منطقتك (الجزائر UTC+1، الصين UTC+8، إلخ). فيما مضى كانت تحفظ "بتوقيتك كأنه UTC" مما أنتج تواريخ مستقبلية → 404. تم إصلاح هذا جذرياً.

## جدولة المقالات للنشر مستقبلاً

يمكنك ضبط تاريخ/وقت مستقبلي لأي مقال — سيُنشر تلقائياً في الموعد.

**في لوحة التحكم:**
عند تحديد تاريخ متقدم، يظهر إنذار ذهبي:
```
🕐 مقال مجدول — سيُعرض كـ "سينشر قريباً" حتى 2026-05-15 14:30 (بعد 3 أيام)
```

**على الموقع:**
- **البطاقة:** شارة ذهبية متوهجة **"سينشر قريباً"**
- **صفحة المقال:** ترحيب جميل بعدّاد تنازلي حي بدل المحتوى:
  - أيقونة ساعة ذهبية متحركة
  - موعد النشر معروض بوضوح
  - عدّاد بالثانية: `يتبقى 3 أيام و 2 ساعة...`
  - زر "العودة للرئيسية"
- عند بلوغ الموعد: الصفحة تُعيد التحميل تلقائياً وتعرض المحتوى الكامل

**ملاحظة:** للجدولة الدقيقة، يجب أن يحدث `git push` بعد بلوغ الموعد (لأن GitHub Pages لا يعيد البناء تلقائياً بدون push). أحد الحلول: إضافة GitHub Actions cron لإعادة البناء يومياً.

## ١٠. إنشاء قسم جديد

### من لوحة التحكم (الأسهل)

1. افتح لوحة التحكم → **الأقسام**
2. اضغط **إنشاء قسم جديد**
3. أدخل:
   - **ID**: معرّف فريد (`latin-lowercase-hyphens` فقط، مثل `hardware`)
   - **الاسم بالعربية**: يظهر في الواجهة العربية
   - **الاسم بالإنجليزية**: يظهر في الواجهة الإنجليزية
   - **الأيقونة**: اختر من منتقي الأيقونات (Font Awesome 6)
   - **اللون**: اختر من لوحة الألوان أو أدخل hex
4. اضغط **إنشاء القسم**

لوحة التحكم تُنشئ تلقائياً:
- `_ar/<id>/` و`_en/<id>/` (مجلدات المقالات)
- `ar/category/<id>.html` و`en/category/<id>.html` (صفحات القسم)
- إدخال في `_data/categories.yml`

### يدوياً

**1. أضف للـ `_data/categories.yml`:**

```yaml
- id: hardware
  name_ar: "الأجهزة والمكونات"
  name_en: "Hardware"
  icon: "fa-solid fa-memory"
  color: "#06b6d4"
```

**2. أنشئ مجلدات المقالات:**

```bash
mkdir -p _ar/hardware _en/hardware
```

**3. أنشئ صفحات القسم:**

`ar/category/hardware.html`:
```yaml
---
layout: category
lang: ar
category: hardware
permalink: /ar/category/hardware/
---
```

`en/category/hardware.html`:
```yaml
---
layout: category
lang: en
category: hardware
permalink: /en/category/hardware/
---
```

بعد `git push`، يظهر القسم تلقائياً في: الهيدر، الشريط، الصفحة الرئيسية، الـ footer — **بدون أي تعديل في القوالب**.
